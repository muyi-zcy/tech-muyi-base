# 08 代码生成器使用说明

本文档基于 `tech-muyi-base-generator` 模块中的 `MyGeneratorCode` 实现整理，面向“首次使用”和“团队标准化使用”场景。

## 1. 生成器定位

`MyGeneratorCode` 是一个 Swing 桌面工具，核心能力：

- 图形化输入数据库与项目参数。
- 从数据库加载表清单（`SHOW TABLES`）。
- 按模板批量生成分层代码。
- 生成前检测同名文件并支持“覆盖确认”。
- 将配置持久化到本地 `~/.tech-muyi-generator-config.json`。

## 2. 生成产物结构（按代码实际路径）

以项目名 `demo-order`、`groupId=com.demo` 为例，生成路径如下：

- `demo-order-client/src/main/java/com/demo/client/query`：`Query`
- `demo-order-client/src/main/java/com/demo/client/dto`：`Dto`
- `demo-order-client/src/main/java/com/demo/client/api`：`ApiService`
- `demo-order-core/src/main/java/com/demo/core/entity`：`Do`
- `demo-order-core/src/main/java/com/demo/core/dao`：`Dao`
- `demo-order-core/src/main/java/com/demo/core/service`：`Service`
- `demo-order-core/src/main/java/com/demo/core/service/impl`：`ServiceImpl`
- `demo-order-core/src/main/java/com/demo/core/service/api/impl`：`ApiServiceImpl`
- `demo-order-core/src/main/java/com/demo/core/manager`：`Manager`
- `demo-order-core/src/main/resources/mapper`：`Mapper`
- `demo-order-web/src/main/java/com/demo/web/controller`：`Controller`

## 3. 启动方式

从 IDE 直接运行：

- 运行类：`tech.muyi.gengrator.MyGeneratorCode`
- 入口方法：`main(String[] args)`

## 4. 页面字段说明（逐项）

### 4.1 基础参数

- `项目名`：生成后的项目根目录名。
- `项目目录`：代码输出父目录（支持浏览选择）。
- `数据库IP` / `端口` / `数据库名`：用于构造 JDBC URL。
- `用户名` / `密码`：数据库连接账号。
- `GroupId`：决定 Java 包路径前缀（会映射为目录层级）。

### 4.2 行为开关

- `覆盖已存在文件`：是否允许覆盖已有文件。
- `生成后不打开目录`：是否关闭生成完成后的目录打开动作（由各模板处理）。
- `多选表`：切换单选/多选模式。

### 4.3 模板开关

可按需勾选：

- `Controller`、`Query`、`Dto`、`Dao`、`Do`
- `ApiService`、`ApiServiceImpl`
- `Service`、`ServiceImpl`
- `Mapper`、`Manager`

## 5. 按钮行为说明（核心方法对应）

### 5.1 测试连接

- 触发方法：`testConnection(url, username, password)`
- 成功提示：`连接成功`
- 失败提示：`连接失败：{异常信息}`

### 5.2 加载表

- 触发方法：`queryTables(url, username, password)`
- 执行 SQL：`SHOW TABLES`
- 行为：清空并重建表列表，按字典序排序。

### 5.3 保存配置

- 触发方法：`saveConfig(GeneratorConfig config)`
- 保存位置：`~/.tech-muyi-generator-config.json`

### 5.4 开始生成

主流程：

1. `buildConfig(...)` 读取页面参数。
2. 校验是否选择了至少一张表。
3. `detectExistingFiles(...)` 检测同名文件。
4. 若未勾选覆盖且有冲突，直接阻断。
5. 若勾选覆盖且有冲突，弹确认框。
6. `generateForTables(...)` 按表和模板批量调用 `*.deal(...)`。

## 6. 表名转类名规则

方法：`toEntityName(String tableName)`。

- 若表名前缀是 `t_` 或 `c_`，会先移除前缀。
- 其余按下划线分段并转驼峰首字母大写。

示例：

- `t_order_item` -> `OrderItem`
- `c_user_account` -> `UserAccount`
- `invoice_detail` -> `InvoiceDetail`

## 7. 推荐标准操作流程（团队版）

1. 先点击“测试连接”。
2. 点击“加载表”并确认表数量。
3. 勾选目标表（建议先单表验证模板）。
4. 勾选所需模板（新模块可全选，增量改造可部分勾选）。
5. 首次生成不勾选“覆盖已存在文件”。
6. 代码检查通过后，再做二次覆盖生成。
7. 生成后立刻补充业务语义与单元测试。

## 8. 常见问题与处理

### 8.1 “请至少选择一张表”

- 未在表列表选中任何表，先执行“加载表”并选择。

### 8.2 “检测到已有文件，且当前未开启覆盖”

- 说明目标目录已有同名文件，选择：
  - 勾选覆盖并二次确认；
  - 或改项目名/路径后重新生成。

### 8.3 “加载表失败”

排查顺序：

1. 数据库账号权限是否允许 `SHOW TABLES`。
2. 数据库名是否正确。
3. IP、端口、防火墙是否可达。

## 9. 生成后必须做的工作（不可省略）

- 补业务校验、异常码、日志上下文。
- 补接口鉴权、租户隔离、幂等控制。
- 补测试（至少 Service 层单测 + 关键接口集成测试）。
- 检查是否符合《代码层级结构规范》。
