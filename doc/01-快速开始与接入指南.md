# 01 快速开始与接入指南

## 1. 项目定位

`tech-muyi-base` 是面向 Spring Cloud Alibaba 体系的企业级后端脚手架，采用多模块 Maven 聚合结构，支持按需引入组件能力。

## 2. 运行环境要求

- JDK: 8 及以上（建议统一到 JDK 8 或 JDK 11 LTS）
- Maven: 3.6+
- Spring Boot: 2.3.12.RELEASE（由父依赖管理）
- 数据库: MySQL 5.7+
- 缓存: Redis 3.0+
- 可选中间件: RocketMQ、Nacos、Zookeeper、MinIO

## 3. 目录与模块概览

常见模块分为以下几类：

- 基础能力：`tech-muyi-base-core`、`tech-muyi-base-common`、`tech-muyi-base-config`、`tech-muyi-base-util`、`tech-muyi-base-exception`、`tech-muyi-base-id`、`tech-muyi-base-on`
- 微服务通信：`tech-muyi-base-rpc`、`tech-muyi-base-feign`、`tech-muyi-base-mq`、`tech-muyi-base-message`、`tech-muyi-base-okhttp`
- 数据存储：`tech-muyi-base-redis`、`tech-muyi-base-oss`、`tech-muyi-base-db-shardingjdbc`、`tech-muyi-base-flyway`
- 治理与安全：`tech-muyi-base-sso`、`tech-muyi-base-sentinel`
- 任务调度：`tech-muyi-base-easyjob`、`tech-muyi-base-elasticjob`
- 可观测：`tech-muyi-base-log`、`tech-muyi-base-tracer`、`my-log-plugins/*`
- 开发辅助：`tech-muyi-base-api`、`tech-muyi-base-generator`、`tech-muyi-base-dmask`

## 4. 接入方式

### 4.1 先选接入形态（重点）

`tech-muyi-base-boot` 与 `tech-muyi-base-dependencies` 都是聚合入口，但定位不同：

- `tech-muyi-base-boot`：偏单体脚手架，默认包含 `core/api/log/config/util/id`。
- `tech-muyi-base-dependencies`：偏 Dubbo 集群脚手架，默认包含 `core/rpc/api/redis/util/id`。

两者不要重复同时引入，建议按架构二选一。

### 4.2 单体服务接入（使用 boot）

```xml
<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>tech-muyi-base-boot</artifactId>
    <version>1.0.0</version>
</dependency>
```

### 4.3 Dubbo 集群服务接入（使用 dependencies）

```xml
<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>tech-muyi-base-dependencies</artifactId>
    <version>1.0.0</version>
</dependency>
```

### 4.4 按需引入功能模块

```xml
<!-- Redis 能力 -->
<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>tech-muyi-base-redis</artifactId>
    <version>1.0.0</version>
</dependency>

<!-- RPC 能力 -->
<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>tech-muyi-base-rpc</artifactId>
    <version>1.0.0</version>
</dependency>
```

### 4.5 application.yml 最小配置（单体）

```yaml
spring:
  application:
    name: demo-service
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://127.0.0.1:3306/demo?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai
    username: root
    password: root
```

### 4.6 application.yml 最小配置（Dubbo 集群）

```yaml
spring:
  application:
    name: demo-rpc-provider
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://127.0.0.1:3306/demo?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai
    username: root
    password: root

dubbo:
  application:
    name: ${spring.application.name}
  registry:
    address: nacos://127.0.0.1:8848
  protocol:
    name: dubbo
    port: -1
```

## 5. 核心方法快速落地（首日必用）

### 5.1 接口统一返回 `MyResult`

```java
@GetMapping("/detail")
public MyResult<OrderDTO> detail(@RequestParam Long id) {
    return MyResult.ok(orderService.getById(id));
}
```

### 5.2 业务异常统一抛 `MyException`

```java
if (order == null) {
    throw new MyException(CommonErrorCodeEnum.PARAM_CHECK_ERROR);
}
```

### 5.3 Redis 常用方法

```java
redissonManage.set("order:1", order, 10, TimeUnit.MINUTES);
OrderDTO cache = redissonManage.get("order:1");
```

### 5.4 RPC 上下文透传

`RpcConsumerContextFilter#invoke()` 会调用 `MyTtlContextManager.downAllData()` 下传上下文；`RpcProviderContextFilter#invoke()` 会调用 `MyTtlContextManager.upAllData()` 恢复上下文。业务侧只要先注册上下文变量并写入值即可透传。
## 6. 推荐接入顺序

1. 第一阶段：仅接入基础模块（core/common/config/exception/util）。
2. 第二阶段：接入数据库与缓存能力（db、redis、flyway）。
3. 第三阶段：接入服务治理（rpc/feign/sentinel/sso）。
4. 第四阶段：接入日志追踪与任务调度（log/tracer/job）。

## 7. 新服务落地清单

上线前建议逐项确认：

- 依赖版本是否全部使用统一父 BOM 管理。
- `MyResult` 返回规范是否统一。
- 全局异常处理是否生效（状态码、错误信息是否可追踪）。
- 链路上下文（traceId/tenant/user）是否可跨线程、跨 RPC 传递。
- 缓存、消息、对象存储是否按环境隔离命名空间。
- Flyway 脚本是否随代码版本发布并可回滚。

## 8. 常见问题

### 8.1 启动时报 Bean 重复定义

- 检查是否同时引入多个功能重复的 starter。
- 检查是否手动创建了与自动配置同名的 Bean。

### 8.2 多环境配置不生效

- 确认 `spring.profiles.active` 与部署环境一致。
- 检查是否有 Nacos 配置覆盖本地配置。

### 8.3 依赖冲突

- 执行 `mvn dependency:tree` 定位版本冲突。
- 优先在父依赖统一收敛版本，不要在业务模块局部随意覆盖。
