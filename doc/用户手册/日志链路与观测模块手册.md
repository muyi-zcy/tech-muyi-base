# 日志链路与观测模块手册

## 1. 模块范围

- `tech-muyi-base-log`
- `tech-muyi-base-tracer`
- `my-log-plugins/*`

## 2. 日志模块（tech-muyi-base-log）

### 2.1 功能定位

- 提供统一日志输出能力，覆盖 API 请求日志与基础审计信息。

### 2.2 关键组件

- `RestApiFilter`：请求过滤与日志记录入口。

### 2.3 核心方法与行为

#### `RestApiFilter#around(ProceedingJoinPoint)`

- 切点：所有 `@RestController`。
- 入口日志：记录 URI、方法名、参数（会跳过 `HttpServletRequest/HttpServletResponse/MultipartFile`）。
- 结果日志：当返回对象为 `MyResult` 时，记录结果和耗时。
- 特殊路径：`/ok` 直接放行，不打印业务日志。

### 2.4 接口日志示例

```java
@GetMapping("/user/detail")
public MyResult<UserDTO> detail(@RequestParam Long userId) {
    return MyResult.ok(userService.detail(userId));
}
```

该接口会自动输出“入参 + 出参 + 耗时”日志，无需在业务代码里手工打重复日志。

### 2.5 日志规范建议

- 必填字段：`traceId`、`serviceName`、`uri`、`method`、`costMs`、`resultCode`。
- 敏感字段脱敏：手机号、身份证、token、密码。
- 大报文截断：限制单条日志最大长度，防止日志爆量。

## 3. 链路追踪模块（tech-muyi-base-tracer）

### 3.1 功能定位

- 提供分布式调用链追踪能力，支持线程内外上下文透传。

### 3.2 接入建议

- Web、RPC、MQ、异步线程统一写入 trace 上下文字段。
- 日志 MDC 与追踪上下文保持一致，便于检索串联。
- 采样率在生产环境按流量分级配置，控制存储成本。

### 3.3 核心方法与行为

#### `MDCSpanExtension`

- `logStartedSpan(Span currentSpan)`：span 开始时把 `traceId/spanId` 写入 MDC。
- `logStoppedSpan(Span currentSpan)`：span 结束时清理 MDC，并回填父 span 的 MDC（若存在）。
- `logStoppedSpanInRunnable(Span currentSpan)`：Runnable 场景结束后清空 MDC，避免线程池串日志。
- `supportName()`：返回扩展名 `slf4jmdc`。

这意味着只要启用该扩展，日志框架就能自动带上 trace 字段。

## 4. my-log-plugins 插件体系

### 4.1 插件列表

- `my-log-core`
- `my-log-springmvc-plugin`
- `my-log-dubbo-plugin`
- `my-log-rocketmq-plugin`
- `my-log-redis-plugin`
- `my-log-db-plugin`
- `my-log-okhttp-plugin`

### 4.2 插件职责

- 针对不同中间件和调用入口进行日志增强。
- 将调用元信息标准化，统一输出格式。

### 4.3 核心类与方法（my-log-core）

#### `MyTracer`

- `buildSpan(String operationName).start()`：创建新 span。
- `activateSpan(Span span)`：激活当前 span，绑定到 scope。
- `activeSpan()`：获取当前线程活跃 span。
- `inject(...)` / `extract(...)`：跨进程注入/提取上下文。
- `log(String event)`：向活跃 span 打日志事件。

#### `MySpan`

- `setTag(...)`：打标签。
- `setBaggageItem(key, value)`：设置跨进程传递字段。
- `log(...)`：写 span 事件日志。
- `finish()`：结束 span，输出 span 信息并刷新 MDC。

### 4.4 使用建议

- 仅按需引入插件，避免不必要的拦截开销。
- 对高频路径（如 Redis、DB）优先做采样和级别控制。
- 关注插件版本兼容，保证与中间件客户端版本匹配。

## 5. 可观测落地模型

建议建立“日志 + 指标 + 链路”三位一体：

1. 日志用于还原业务上下文。
2. 指标用于发现异常趋势和容量瓶颈。
3. 链路用于快速定位跨服务耗时热点。

## 6. 常见问题与排查

### 6.1 traceId 丢失

- 检查线程池是否启用上下文透传装饰器。
- 检查 RPC/MQ 过滤器是否注入和提取上下文。

### 6.2 日志量过大

- 调整日志级别与采样策略。
- 对重复请求和健康检查路径做降噪处理。

### 6.3 检索困难

- 统一 JSON 日志格式与字段命名。
- 为核心字段建立固定检索模板。
