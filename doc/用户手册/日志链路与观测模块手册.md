# 日志链路与观测模块手册

## 1. 模块范围

- `tech-muyi-base-log`：统一日志配置
- `my-log-plugins/*`：基于 SOFATracer 的链路追踪插件体系

## 2. 日志模块（tech-muyi-base-log）

### 2.1 功能定位

- 提供统一日志输出能力，覆盖 API 请求日志与基础审计信息。
- 集成 SOFATracer MDC 字段，自动关联 traceId 和 spanId。

### 2.2 关键组件

- `RestApiFilter`：请求过滤与日志记录入口。
- Logback 配置：支持多环境（local/dev/pre/prod）日志输出。

### 2.3 核心方法与行为

#### `RestApiFilter#around(ProceedingJoinPoint)`

- 切点：所有 `@RestController`。
- 入口日志：记录 URI、方法名、参数（会跳过 `HttpServletRequest/HttpServletResponse/MultipartFile`）。
- 结果日志：当返回对象为 `MyResult` 时，记录结果和耗时。
- 特殊路径：`/ok` 直接放行，不打印业务日志。

### 2.4 接口日志示例

```java
@GetMapping(“/user/detail”)
public MyResult<UserDTO> detail(@RequestParam Long userId) {
    return MyResult.ok(userService.detail(userId));
}
```

该接口会自动输出”入参 + 出参 + 耗时”日志，无需在业务代码里手工打重复日志。

### 2.5 日志格式与 TraceId

所有日志自动包含 SOFATracer 的 traceId 和 spanId：

```
2024-05-05 10:30:45.123 my-service, [0a1b2c3d4e5f6g7h] [1.1] [http-nio-8080-exec-1] INFO  c.e.UserController - 用户查询请求
```

- `[0a1b2c3d4e5f6g7h]`：traceId，全链路唯一标识
- `[1.1]`：spanId，当前调用节点标识

### 2.6 日志规范建议

- 必填字段：`traceId`、`serviceName`、`uri`、`method`、`costMs`、`resultCode`。
- 敏感字段脱敏：手机号、身份证、token、密码。
- 大报文截断：限制单条日志最大长度，防止日志爆量。

## 3. 链路追踪模块（my-log-plugins）

### 3.1 技术选型

基于 **SOFATracer 3.1.1** 构建，提供分布式调用链追踪能力。

**为什么选择 SOFATracer？**
- 阿里开源，生产级稳定性
- 支持主流中间件官方插件
- 自动 MDC 集成，日志天然带 traceId
- 低侵入，性能开销小

### 3.2 插件体系架构

```
my-log-plugins/
├── my-log-core                  # 核心配置与工具类
├── my-log-springmvc-plugin      # SpringMVC 请求追踪（官方插件）
├── my-log-dubbo-plugin          # Dubbo RPC 追踪（官方插件）
├── my-log-feign-plugin          # Feign 客户端追踪（官方插件）
├── my-log-okhttp-plugin         # OkHttp 客户端追踪（官方插件）
├── my-log-db-plugin             # 数据库 SQL 追踪（官方插件）
├── my-log-redis-plugin          # Redis 操作追踪（官方插件）
└── my-log-rocketmq-plugin       # RocketMQ 消息追踪（自定义实现）
```

**插件分类：**
- **官方插件 Wrapper**：SpringMVC、Dubbo、Feign、OkHttp、DataSource、Redis
  - 使用 SOFATracer 官方插件，提供统一配置入口
- **自定义实现**：RocketMQ
  - 基于 SOFATracer API 自行实现 Hook

### 3.3 快速开始

#### 3.3.1 引入依赖

```xml
<!-- 核心模块（必选） -->
<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>my-log-core</artifactId>
</dependency>

<!-- 按需引入插件 -->
<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>my-log-springmvc-plugin</artifactId>
</dependency>

<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>my-log-dubbo-plugin</artifactId>
</dependency>

<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>my-log-feign-plugin</artifactId>
</dependency>

<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>my-log-db-plugin</artifactId>
</dependency>
```

#### 3.3.2 配置文件

```yaml
# application.yml
tech:
  muyi:
    tracer:
      enabled: true                    # 全局开关
      log-path: ./logs                 # 日志路径
      log-reserve-days: 7              # 日志保留天数
      sampling-rate: 1.0               # 采样率（0.0-1.0）
      
      # 各插件独立开关
      springmvc:
        enabled: true
      dubbo:
        enabled: true
      feign:
        enabled: true
        log-request-body: false        # 是否记录请求体
        log-response-body: false       # 是否记录响应体
        max-request-body-length: 1024  # 请求体最大长度
        max-response-body-length: 1024 # 响应体最大长度
      db:
        enabled: true
        log-parameters: true           # 是否记录 SQL 参数
        max-parameter-length: 1000     # 参数最大长度
      redis:
        enabled: true
      rocketmq:
        enabled: true
```

#### 3.3.3 自动生效

配置完成后，所有 HTTP 请求、RPC 调用、数据库操作、Redis 访问都会自动追踪，无需修改业务代码。

### 3.4 核心工具类（my-log-core）

#### `TraceUtils`

```java
// 获取当前 TraceId
String traceId = TraceUtils.getCurrentTraceId();

// 获取当前 SpanId
String spanId = TraceUtils.getCurrentSpanId();

// 判断是否在追踪上下文中
boolean inTrace = TraceUtils.isInTraceContext();

// 格式化参数（限制长度）
String params = TraceUtils.formatParameters(paramList, 1000);

// 格式化异常堆栈（限制长度）
String stackTrace = TraceUtils.formatStackTrace(exception, 2000);
```

### 3.5 各插件详细说明

#### 3.5.1 SpringMVC 插件

**追踪内容：**
- HTTP 请求路径、方法、参数
- 响应状态码、耗时
- 异常信息

**自动生效条件：**
- 项目中存在 `@RestController` 或 `@Controller`

**日志输出：**
```
springmvc-digest.log    # 摘要日志
springmvc-stat.log      # 统计日志
```

#### 3.5.2 Dubbo 插件

**追踪内容：**
- 服务接口、方法名
- 请求参数、响应结果
- 调用耗时、异常信息

**支持版本：**
- Dubbo 3.x（推荐）
- Dubbo 2.7.x

**日志输出：**
```
dubbo-client-digest.log    # 客户端摘要
dubbo-client-stat.log      # 客户端统计
dubbo-server-digest.log    # 服务端摘要
dubbo-server-stat.log      # 服务端统计
```

#### 3.5.3 Feign 插件

**追踪内容：**
- HTTP 请求 URL、方法
- 请求头、请求体（可配置）
- 响应状态码、响应体（可配置）
- 调用耗时、异常信息

**自动生效条件：**
- 项目中使用 Spring Cloud OpenFeign
- 存在 `@FeignClient` 注解的接口

**配置项：**
```yaml
tech:
  muyi:
    tracer:
      feign:
        enabled: true
        log-request-body: false        # 记录请求体
        log-response-body: false       # 记录响应体
        max-request-body-length: 1024  # 请求体最大长度
        max-response-body-length: 1024 # 响应体最大长度
```

**日志输出：**
```
feign-digest.log    # Feign 调用摘要
feign-stat.log      # Feign 统计
```

**使用示例：**
```java
@FeignClient(name = "user-service", url = "http://localhost:8080")
public interface UserClient {
    @GetMapping("/api/user/{id}")
    UserDTO getUser(@PathVariable Long id);
}

// 调用时自动追踪，无需额外配置
UserDTO user = userClient.getUser(123L);
```

#### 3.5.4 DataSource 插件（my-log-db-plugin）

**追踪内容：**
- SQL 语句
- 绑定参数（可配置）
- 执行耗时
- 影响行数

**覆盖范围：**
- 所有 JDBC DataSource（包括 Druid、HikariCP）
- ShardingSphere 分片后的 SQL
- MyBatis/MyBatis-Plus 执行的 SQL

**配置项：**
```yaml
tech:
  muyi:
    tracer:
      db:
        log-parameters: true           # 记录 SQL 参数
        max-parameter-length: 1000     # 参数最大长度
        log-resultset-size: false      # 记录结果集大小
```

**日志输出：**
```
datasource-client-digest.log    # SQL 执行摘要
datasource-client-stat.log      # SQL 统计
```

#### 3.5.5 Redis 插件

**追踪内容：**
- Redis 命令
- 执行耗时
- 异常信息

**支持客户端：**
- Jedis
- Lettuce

**日志输出：**
```
redis-digest.log    # Redis 操作摘要
redis-stat.log      # Redis 统计
```

#### 3.5.6 RocketMQ 插件

**追踪内容：**
- 消息发送：topic、tags、broker、msgId
- 消息消费：topic、tags、消费组、消息数量
- 执行耗时、异常信息

**实现方式：**
- 基于 RocketMQ Hook 机制
- 自动注入 traceId 到消息属性

**配置示例：**
```java
// 生产者自动注入 Hook
DefaultMQProducer producer = new DefaultMQProducer(“producer-group”);
// 无需手动配置，插件自动注册

// 消费者自动注入 Hook
DefaultMQPushConsumer consumer = new DefaultMQPushConsumer(“consumer-group”);
// 无需手动配置，插件自动注册
```

#### 3.5.7 OkHttp 插件

**追踪内容：**

### 3.6 链路追踪日志

SOFATracer 会为每个插件生成两类日志：

**1. 摘要日志（digest.log）**
- 记录每次调用的详细信息
- 包含 traceId、spanId、耗时、状态等
- 用于问题排查和链路还原

**2. 统计日志（stat.log）**
- 按时间窗口聚合统计
- 包含调用次数、成功率、平均耗时等
- 用于性能监控和容量规划

**日志路径：**
```
${tech.muyi.tracer.log-path}/
├── springmvc-digest.log
├── springmvc-stat.log
├── dubbo-client-digest.log
├── dubbo-server-digest.log
├── datasource-client-digest.log
├── redis-digest.log
└── ...
```

### 3.7 采样策略

**全量采样（默认）：**
```yaml
tech:
  muyi:
    tracer:
      sampling-rate: 1.0    # 100% 采样
```

**按比例采样：**
```yaml
tech:
  muyi:
    tracer:
      sampling-rate: 0.1    # 10% 采样
```

**建议：**
- 开发/测试环境：1.0（全量）
- 生产环境：0.1-0.3（根据流量调整）
- 核心链路：1.0（全量）
- 健康检查：0（关闭）

### 3.8 异步线程追踪

SOFATracer 自动支持异步线程上下文传递（基于 TransmittableThreadLocal）：

```java
@Async
public void asyncTask() {
    // traceId 自动传递到异步线程
    String traceId = TraceUtils.getCurrentTraceId();
    log.info(“异步任务执行，traceId: {}”, traceId);
}
```

**线程池配置：**
```java
@Bean
public Executor taskExecutor() {
    ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
    executor.setCorePoolSize(10);
    executor.setMaxPoolSize(20);
    // SOFATracer 会自动装饰线程池
    return executor;
}
```

## 4. 可观测落地模型

建议建立”日志 + 指标 + 链路”三位一体：

1. **日志**：用于还原业务上下文
   - 业务日志自动带 traceId
   - 通过 traceId 串联全链路日志

2. **指标**：用于发现异常趋势和容量瓶颈
   - 从 stat.log 提取指标数据
   - 监控 QPS、耗时、错误率

3. **链路**：用于快速定位跨服务耗时热点
   - 从 digest.log 还原调用链
   - 分析各节点耗时占比

## 5. 最佳实践

### 5.1 按需引入插件

只引入实际使用的插件，避免不必要的性能开销：

```xml
<!-- 只引入 Web + Feign + DB 插件 -->
<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>my-log-springmvc-plugin</artifactId>
</dependency>
<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>my-log-feign-plugin</artifactId>
</dependency>
<dependency>
    <groupId>tech.muyi</groupId>
    <artifactId>my-log-db-plugin</artifactId>
</dependency>
```

### 5.2 生产环境优化

```yaml
tech:
  muyi:
    tracer:
      sampling-rate: 0.1              # 降低采样率
      log-reserve-days: 3             # 缩短日志保留
      feign:
        log-request-body: false       # 关闭请求体记录
        log-response-body: false      # 关闭响应体记录
      db:
        log-parameters: false         # 关闭参数记录
        log-resultset-size: false     # 关闭结果集记录
```

### 5.3 敏感信息处理

```java
// 在业务日志中避免直接打印敏感信息
log.info(“用户登录，userId: {}”, userId);  // ✓
log.info(“用户登录，password: {}”, password);  // ✗
```

### 5.4 健康检查路径排除

```yaml
tech:
  muyi:
    tracer:
      springmvc:
        exclude-patterns:
          - /actuator/**
          - /health
          - /ok
```

## 6. 常见问题与排查

### 6.1 traceId 丢失

**现象：**
- 日志中 traceId 为空或断链

**排查：**
1. 检查 my-log-core 是否正确引入
2. 检查对应插件是否启用
3. 检查异步线程是否使用 @Async 或装饰过的线程池

**解决：**
```yaml
tech:
  muyi:
    tracer:
      enabled: true    # 确保全局开关打开
```

### 6.2 日志量过大

**现象：**
- 磁盘占用快速增长
- 日志文件过多

**排查：**
1. 检查采样率配置
2. 检查是否记录了大量健康检查请求
3. 检查 SQL 参数是否过长

**解决：**
```yaml
tech:
  muyi:
    tracer:
      sampling-rate: 0.1              # 降低采样率
      log-reserve-days: 3             # 缩短保留时间
      db:
        log-parameters: false         # 关闭参数记录
        max-parameter-length: 500     # 限制参数长度
```

### 6.3 性能影响

**现象：**
- 接口响应变慢
- CPU 使用率上升

**排查：**
1. 检查是否全量采样
2. 检查是否记录了大量 SQL 参数
3. 检查日志写入是否成为瓶颈

**解决：**
```yaml
tech:
  muyi:
    tracer:
      sampling-rate: 0.1              # 降低采样率
      db:
        log-parameters: false         # 关闭参数记录
```

### 6.4 插件不生效

**现象：**
- 引入插件后没有生成追踪日志

**排查：**
1. 检查插件依赖是否正确引入
2. 检查对应中间件客户端是否存在
3. 检查插件开关是否启用

**解决：**
```yaml
tech:
  muyi:
    tracer:
      enabled: true
      springmvc:
        enabled: true    # 确保插件开关打开
      feign:
        enabled: true    # 确保 Feign 插件开关打开
```
