## 2. RPC 模块（Dubbo）

### 2.1 功能定位

- 封装 Dubbo 调用链上下文透传和基础过滤能力。

### 2.2 关键组件

- `RpcConsumerContextFilter`：消费者侧上下文注入。
- `RpcProviderContextFilter`：提供者侧上下文提取与清理。

### 2.3 核心方法与调用流程

#### `RpcConsumerContextFilter#invoke(Invoker<?>, Invocation)`

- 判断 consumer 侧调用。
- 调用 `MyTtlContextManager.downAllData()` 读取当前线程全部 TTL 上下文。
- 使用 `RpcContext.setAttachments(...)` 下传到 RPC 请求。
- 最终执行 `invoker.invoke(invocation)`。

#### `RpcProviderContextFilter#invoke(Invoker<?>, Invocation)`

- 判断 provider 侧调用。
- 调用 `MyTtlContextManager.upAllData(rpcContext.getAttachments())` 恢复上下文。
- 记录 RPC 入参与耗时日志。
- 执行目标服务并输出结果日志。

### 2.4 使用建议

- 透传字段建议固定：`traceId`、`tenantId`、`userId`。
- 过滤器内避免重逻辑，保障调用链低开销。

### 2.5 业务侧示例（透传 requestId）

```java
// 发起 RPC 前
REQUEST_ID_TTL.set("req-20260427001");
// 随后正常发起 Dubbo 调用，过滤器会自动透传
```

## 3. Feign 模块

### 3.1 功能定位

- 提供 OpenFeign 统一接入能力，适用于 HTTP 服务间调用。

### 3.2 接入建议

- 为每个 Feign Client 设置合理超时和重试策略。
- 对下游错误统一映射为平台错误码，避免原样透出内部异常。
- 对幂等接口可加重试，对非幂等接口禁用自动重试。

### 3.3 核心方法建议

- 优先在 Feign 的请求拦截器里注入 `traceId/tenantId`，不要在业务调用点逐个拼 header。
- 错误解码器统一实现 `ErrorDecoder#decode`，将下游错误码映射到平台异常模型。

## 4. MQ 模块（RocketMQ）

### 4.1 功能定位

- 封装消息生产消费基础能力和错误码规范。

### 4.2 关键对象

- `MqErrorCodeEnum`：MQ 相关错误语义统一。

### 4.3 使用建议

- Topic 与 Tag 命名采用“业务域 + 事件”规范。
- 消费端必须实现幂等（去重表/状态机/CAS）。
- 失败重试和死信队列策略要在业务侧明确。

## 5. Message 模块

### 5.1 功能定位

- 提供消息发送基础能力（如邮件通知等）。

### 5.2 使用建议

- 模板化消息内容，避免业务代码硬编码文案。
- 将发送动作异步化，提升主链路响应性能。

## 6. SSO 模块

### 6.1 功能定位

- 提供单点登录与多租户基础支持能力。

### 6.2 关键组件

- `MySsoManager`：SSO 核心管理能力。
- `SsoInterceptor`：请求拦截与身份检查。
- `MyTenantLineInnerInterceptor`：多租户 SQL 拦截。
- `SsoInfoHook`、`SsoInfoHookFactory`：扩展点机制。

### 6.3 使用建议

- 租户隔离字段必须全链路透传并在 SQL 层校验。
- 对白名单接口单独配置，避免误拦截基础健康检查接口。

## 7. Sentinel 模块

### 7.1 功能定位

- 提供限流、熔断、降级的治理能力接入。

### 7.2 使用建议

- 先做监控阈值，再开启硬限流，避免一次性误杀流量。
- 熔断降级返回应统一成业务可识别的错误结构。

## 8. OkHttp 模块

### 8.1 功能定位

- 对外 HTTP 调用基础封装，适用于轻量第三方接口访问。

### 8.2 使用建议

- 连接池大小和超时时间按业务并发模型调优。
- 对外调用日志需脱敏（token、手机号、身份证等）。

## 9. 通信链路统一实践

1. 上下文统一：trace/tenant/user 全链路透传。
2. 错误统一：所有远程异常映射到平台错误码体系。
3. 超时统一：接口、消息、重试策略按分级标准配置。
4. 容错统一：幂等、降级、隔离、限流形成闭环。
