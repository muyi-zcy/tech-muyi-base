# 03 数据与存储模块手册

## 1. 模块范围

- `tech-muyi-base-redis`

- `tech-muyi-base-flyway`

## 2. Redis 模块（tech-muyi-base-redis）

### 2.1 功能定位

- 提供基于 Redisson 的缓存、分布式锁、集合对象操作能力。
- 统一 Redis JSON 编解码策略，降低序列化兼容风险。

### 2.2 关键组件

- `RedissonManageAutoConfiguration`：自动装配入口。
- `RedissonManage`：封装常用 Redis 操作能力。
- `CustomJsonJacksonCodec`：统一 JSON 序列化策略。

### 2.3 `RedissonManage` 核心方法（重点）

#### 锁相关

- `lock(String lockKey)`：阻塞获取锁，默认看门狗续期。
- `lock(String lockKey, long leaseTime)`：带租期锁（秒）。
- `tryLock(String lockKey)`：非阻塞抢锁，返回是否成功。
- `tryLock(String lockKey, TimeUnit unit, long waitTime, long leaseTime)`：限时抢锁并返回 `RLock`。
- `unlock(String lockKey)` / `unlock(RLock lock)`：仅当前线程持有时解锁。

#### KV 相关

- `set(String key, Object value)`：写入常驻值。
- `set(String key, Object value, long expire, TimeUnit unit)`：写入并设置 TTL。
- `get(String key)`：读取对象。
- `delete(String key)`：删除键。

#### List 相关

- `addListElement(String key, Object value)` / `addListAllElement(String key, Collection value)`。
- `getList(String key)` / `getListOneElement(String key, int index)`。
- `removeListOneElement(String key, Object value)` / `deleteList(String key)`。

#### 计数与布隆过滤器

- `incrby(String key)` / `incrby(String key, Long delta)`：原子递增。
- `incrby(String key, Long delta, Long expire, TimeUnit unit)`：递增并设置过期。
- `decrby(String key)`：原子递减。
- `initBloomFilter(String key, Long expectedInsertions, double falseProbability)`：初始化布隆过滤器。
- `addIndexBloomFilter(String key, T value)` / `containIndexBloomFilter(String key, T value)`：添加与判断。

### 2.4 推荐配置示例

```yaml
spring:
  redis:
    host: 127.0.0.1
    port: 6379
    password:
    timeout: 3000
```

### 2.5 代码示例（分布式锁）

```java
RLock lock = redissonManage.tryLock("order:pay:" + orderId, TimeUnit.SECONDS, 2, 15);
if (lock == null || !lock.isHeldByCurrentThread()) {
    throw new MyException(RedisErrorCodeEnum.REDIS_LOCK_ERROR);
}
try {
    // 执行业务逻辑
} finally {
    redissonManage.unlock(lock);
}
```

### 2.6 使用建议

- Key 命名加环境和业务前缀，如 `prod:order:detail:{id}`。
- 分布式锁使用 `tryLock` 时必须设置等待与过期时间，防止死锁。
- 复杂对象缓存时，注意字段变更的向后兼容。

### 2.7 常见问题

- 序列化失败：检查对象是否可序列化以及 codec 是否统一。
- 锁误删：仅释放当前线程持有的锁，异常流程注意 finally 兜底。

## 3. 分库分表模块（tech-muyi-base-db-shardingjdbc）

### 3.1 功能定位

- 提供 Sharding-JDBC 相关配置和健康检测支持。

### 3.2 关键组件

- `DataSourceHealthConfig`：数据源健康状态治理。
- `LocalDateTimeTypeHandler`：时间类型兼容处理。

### 3.3 接入建议

- 优先在测试环境压测分片规则，避免路由热点。
- 广播表、绑定表、分片键设计要在建模阶段确定。
- SQL 尽量包含分片键，减少全路由扫描。

## 4. Flyway 模块（tech-muyi-base-flyway）

### 4.1 功能定位

- 提供数据库版本迁移自动化能力。

### 4.2 关键组件

- `FlywayAutoConfiguration`：迁移自动配置。
- `FlywayDataSourceProperties`：迁移数据源属性。

### 4.3 目录规范建议

- 脚本命名：`V{版本号}__{说明}.sql`，如 `V1_0_1__init_table.sql`
- 一次发布只追加，不修改历史已执行脚本。
- 结构变更与数据修复分开脚本维护，便于回滚评估。

## 5. 数据模块组合建议

- 基础推荐：MySQL + Flyway + Redis。
- 复杂场景：高并发订单类业务可叠加分库分表。
- 文件场景：图片/附件业务请直接查看 `doc/12-oss模块使用说明.md`。
